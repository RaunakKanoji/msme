## Feature completion rules
A feature can move to Demo Ready only when:
1. route/component exists,
2. backend API exists or mock adapter is explicit,
3. data model is represented,
4. tests cover happy and failure path,
5. audit events exist if decision-impacting,
6. accessibility and empty/error states exist,
7. docs are updated.
